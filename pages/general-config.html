    Sync Conflicts

    How to use this dialog ▸

    Current = your active app data folder.  Sync folder = the saved sync location you configured.
    Pick an action for each file, then click Apply. Use the bulk buttons at the top to set all rows at once.

    Conflict rows — both sides changed since last sync

    Keep newerCompares file dates. The more recently modified version overwrites the older one on the other side. The older file is lost.
    Keep currentYour app data wins. The sync folder copy is overwritten. The sync version is lost.
    Keep sync folderThe sync folder wins. Your app data copy is overwritten. The current version is lost.
    SkipBoth files are left exactly as they are. The conflict will reappear on the next sync.
    Archive app (.bak)Renames your app file to a timestamped .bak (e.g. config.js.2026-06-26T14-30.bak), then copies the sync version into the app. The app now has the sync version; the old app version is preserved as a backup.
    Archive sync (.bak)Renames the sync file to a timestamped .bak, then copies your app version to sync. The sync folder now has the app version; the old sync version is preserved as a backup.
    Archive older (.bak)Compares file dates. The older file is renamed to a .bak and the newer version replaces it on both sides. If both files have the exact same date, both are archived and neither wins.
    Archive both (.bak)Both files are renamed to .bak. Neither version remains active. Use when you want to discard both conflicting versions and start fresh.

    New file rows — file only exists on one side New

    Copy to sync(App-only files) Copies your app file to the sync folder so both sides have it.
    Copy to app(Sync-only files) Copies the sync file into your app data so both sides have it.
    Delete (app)Permanently deletes the file from your app data. Cannot be undone.
    Delete (sync)Permanently deletes the file from the sync folder. Cannot be undone.
    Archive app (.bak)Renames the app file to a .bak. The file is removed from active use but preserved on disk.
    Archive sync (.bak)Renames the sync file to a .bak. The file is removed from the sync folder but preserved on disk.
    Archive both (.bak)Archives whichever side has the file (since only one exists here, the effect is the same as archiving that side).
    SkipLeaves the file where it is. It will appear again on the next sync.

    Bulk buttons

    Keep all currentSets every row to "Keep current". App data wins everywhere.
    Keep all sync folderSets every row to "Keep sync folder". Sync data wins everywhere.
    Skip allSets every row to "Skip". Nothing is changed; conflicts remain for next time.
    Archive olderSets all conflict rows (both sides changed) to "Archive older (.bak)". New-file rows are not affected since they don't have that option.
    Archive allSets every row to "Archive both (.bak)". All conflicting files on both sides are backed up; none remain active.

    Backup files are named filename.YYYY-MM-DDTHH-MM.bak and accumulate over time. Use Data & Storage → Manage backup files to review and clean them up.
