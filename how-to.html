Overview

Class Management Tools is a suite of classroom tools for managing students, running activities, tracking grades, and presenting vocabulary exercises. Each tool opens in its own window and runs simultaneously.

Tip: Open and side by side — the Board mirrors the timer and sync state from Class Management in real time. Configure this in .
🛠

First Setup

Follow these six steps the first time you install or launch Class Management Tools. Each step only takes a few minutes, and you only need to do it once.

All your data — class groups, word banks, grades, and session logs — is stored in a single folder on your computer. On first launch you will be asked where to put it.

  1. When the app starts for the first time a blue First Launch banner appears at the top of the launcher. If you are migrating from a previous installation, click Select Existing Folder and point to your old data folder. Otherwise click Start Fresh.
  2. The default location is next to the app itself (or inside your home folder on Windows). You can change it any time in .
  3. If you want the data folder on a USB stick, an external drive, or a cloud-synced folder (Dropbox, OneDrive), click Change folder… in Data & Storage and point to that location.
  4. Optionally set a Backup folder — any save will copy files there too. Ideal for a cloud drive.
  5. Optionally set a Sync folder — useful when you switch between two computers (e.g., classroom PC and a laptop at home).
Recommendation: Set the data folder to a cloud-synced folder (OneDrive, Google Drive, Dropbox) from the start. You get automatic backup and cross-device sync for free.

Class groups are the foundation of every other tool. Set them up once here; they will be available everywhere — Class Management, Participation Tracker, Grade Sheet, and Learning Tools.

  1. Open or to manage your class groups.
  2. Click New Group, give it a name (e.g., "3B English"), and add students one by one or paste a list.
  3. To create a half-group (e.g., for split classes), add a group with the same base name and mark it as a half-group. Half-groups are automatically excluded from random pickers when the full group is loaded.
  4. Repeat for each class you teach. You can create as many groups as you need.
Tip: You can also import a student list from a CSV file — one name per line — using the Import button in the group editor.

Setting the active term tells every tool which school year and term period you are currently in. It powers the end-of-term archive prompt, the Participation Tracker's This Term date filter, and the default values when you create new groups.

  1. Open . At the top you will see the Active Context banner.
  2. Enter the Year (e.g. 2025-2026), select Term (S1 or S2), and fill in the Start date and End date for the term.
  3. If you also use the , click ↕ Planner to pull the dates from a configured Planner term in one click.
  4. Click Save. The active context is now shared with all other tools.
Tip: When the term end date passes and you still have active groups, Group Editor will offer to archive the whole term at once. Archived groups are hidden from rosters but their data is preserved.

Learning Tools games are powered by your content library. You can start using Class Management right away without any content, but activities like Flash Cards, Hangman, or Dictation need content first.

  1. Open and click Word Banks.
  2. Click New Theme and name it after the unit or topic (e.g., "Unit 1 — Introductions").
  3. Add words. Each word can carry translations in up to four languages — French, English, German, and Italian — plus definition, IPA, and synonyms. The richer each entry is, the more game types unlock for that theme.
  4. If you run oral dictations, click Dictations and create a sentence set linked to the theme.
  5. For grammar exercises, go to Grammar Banks and create gap-fill item sets.
Tip: You do not need to fill everything in one go. Start with 10–15 words for a unit and add more over the school year. The games auto-adjust to whatever is in the bank.

A few settings make the app feel like yours. Open from the launcher.

  1. App Identity — Set the title to your school name or your name. It appears in the launcher header.
  2. Language — Choose the interface language (English, French, German, or Italian). This also sets which translation column is shown by default in vocabulary activities.
  3. Startup & Launch — Choose which tools open automatically when the launcher starts, and how they arrange on screen. The most common setup is Split screen with Class Management on the left and Board on the right — both open side by side instantly every time.
  4. Per-app Notes — Jot any reminders for each tool; notes appear on the launcher card so you see them before opening.

With groups and content in place, here is the typical flow for a class session.

  1. Open the launcher and click Class Management (or let it open automatically if you set up startup apps).
  2. Select your class group from the dropdown at the top. The student list loads instantly.
  3. Click Pick to randomly call on a student, then award or deduct points based on their answer.
  4. Open the Board on your projector screen — constellation diagrams, the timer, and notes will be visible to students without exposing the class controls.
  5. At the end of class, click Save Session in Class Management — scores, badges, and notes are saved to the history log.
  6. Open any time during or after class for vocabulary and grammar activities powered by your word banks.
Next steps: Once you are comfortable with the basics, explore , for test recording, and for long-term analysis.

Board

A free-form canvas designed to be projected for the class. Build constellation (mind map) diagrams, draw, add images, and display a live timer synced from .

Build visual word networks with nodes, links, annotations, and colour coding. Content auto-enriched via Wiktionary.

  1. Click an empty area of the canvas to add a new word node, type the word, then press Enter.
  2. Right-click any node to open the context menu — add synonyms, antonyms, a definition, IPA annotation, or French translation.
  3. Click + in the context menu to add a connected word; choose arrow direction (→, ←, ↔) or no arrow.
  4. Select a node and use the format panel to change shape (oval, circle, square, rectangle), border style, and colour.
  5. Use Group to assign nodes to a colour group for visual clustering.
  6. Export: click 🖨 Print, then choose which columns (translation, IPA, definition…) to include in the PDF.
Tip: Use DEF in the node context menu to auto-fetch a definition, or FR for a French translation via Wiktionary.

  1. Click 🧩 Media → 🎙 Record Audio, or right-click an empty area of the board canvas and choose 🎙 Record Audio.
  2. The recorder popup shows a mic level bar and a live timer. Press ⏺ to start recording.
  3. Press ⏹ to stop. A preview player appears so you can listen back before saving.
  4. Click Save to board — the clip is saved in the constellation's companion folder (audio/) and placed on the board as a sound node automatically.
No constellation open? A 💾 Save constellation & proceed button appears. Saving your board first creates the companion folder required to store recordings.

Trimming a recording: right-click any audio node and choose ✂ Trim. Drag the blue handles on the waveform to mark the start and end of the portion you want to keep, preview it with ▶ Selection, then click Save trimmed. A new .wav file is saved alongside the original — the original is never deleted.

Group Editor

Create, edit, and archive your class groups. Groups defined here are shared across every tool — Class Management, Participation Tracker, Grade Sheet, and Learning Tools all read from the same file.

  1. Enter a Group name (e.g. 3B English), select the Term (S1 or S2), and optionally fill Year and Level.
  2. Paste or type student names — one per line or comma-separated — in the Students field, then click Add.
  3. To rename a student or group, edit the field inline and click Save.
  4. To delete a group, click the ✕ button on its row. This cannot be undone from within the editor; use Archive if you want to preserve the data.
  5. Click 💾 Save at the bottom to write all changes to class-groups.js. Changes are not applied until you save.
Tip: You can import groups from a CSV file using the Import CSV button in the toolbar. Columns: Group, First Name, Last Name, Year, Term, Level.

The Active Context banner at the top of Group Editor sets the default year, term, and dates used by every tool. Setting it once here propagates to Class Management, Participation Tracker, and Grade Sheet.

  1. Set the Year field (e.g. 2025-2026) and select the current Term — S1 or S2.
  2. Enter the Start date and End date for the term. These power the Participation Tracker's This Term filter and the end-of-term archive prompt.
  3. If you use the Planner, click ↕ Planner to auto-fill the dates from a configured Planner term. If the Planner has one term, it applies immediately; if it has several, a picker appears.
  4. Click 💾 Save to persist the active context to class-groups.js.

Archiving hides groups from all rosters without deleting their data. Participation files are moved to an archived/ sub-folder. You can unarchive at any time.

  1. To archive a single group, click the Archive button on its row.
  2. To archive all groups for a term at once, click Archive S1 or Archive S2 in the section header.
  3. When the active term end date has passed and active groups remain, a yellow banner appears at the top offering to archive the whole term in one click.
  4. To view or restore archived groups, scroll to the Archived Groups section and click Unarchive on any row.
Note: Archiving a group marks it archived: true in class-groups.js and moves its participation files to archived/. Grade Sheet data is not affected.

Planner

Plan your term week by week — schedule lessons, tests, assignments, and holidays per class. Export to ICS (Google/Apple Calendar), PDF, CSV, or Markdown.

  1. Click + Term in the toolbar to create a new term. Give it a label (e.g. Autumn 2025).
  2. Enter the Start date and End date. If Group Editor already has active-context dates, a blue hint appears in the modal — click it to fill the dates automatically.
  3. Add Holidays within the term (school breaks, bank holidays). Entries scheduled on holiday days are flagged in the week view.
  4. Click OK to save. A week-by-week calendar for that term is now generated.
  5. Use the term selector in the toolbar to switch between terms.
Tip: Term dates set in the Planner can be synced back to Group Editor using the ↕ Planner button in the Active Context banner.
  1. Click Classes in the toolbar to open the class setup panel.
  2. Select a class from the dropdown — the list is drawn from your groups in Group Editor.
  3. Set a colour for the class so its entries are visually distinct in the week view.
  4. Assign the class to Term 1, Term 2, or both.
  5. Define a Weekly Schedule — which days and times the class meets. The planner uses this to pre-populate the week view and warn about scheduling conflicts.
  6. Add Learning Objectives for the term — these can be attached to individual lesson entries.
  1. Click any cell in the week view, or click + Add in the toolbar, to open the entry modal.
  2. Choose the Class, Date, and Type — Lesson, Test, Assignment Due, or Holiday (plus any custom types you have added).
  3. Fill in optional fields: Topic, Objective, Readings, Activities, Homework, and Notes.
  4. For Test entries linked to Grade Sheet: if a Grade Sheet class with the same name exists and has a free test slot, the test is automatically added there. A green badge confirms the link.
  5. Use Duplicate to copy an entry to another date — useful for repeating lesson structures.
  1. Click Export in the toolbar and choose a format.
  2. ICS — exports a calendar file you can import into Google Calendar, Apple Calendar, or Outlook. Optionally filter by class.
  3. PDF — a print-ready version of the week view for the selected term.
  4. CSV — a spreadsheet of all entries with full field data.
  5. Markdown — a structured text document, useful for pasting into a notes app or sharing as plain text.

Manage Database

The content library powering . Nine content types are supported: Word Banks, Quotes, Dictations, Grammar Banks, Gap Fill, Error Banks, Sentences, Stories, and Quizzes. Select a type in the left sidebar, open or create a file, then add, edit, or delete records. Changes are live immediately in all activities.

  1. Click Word Banks and create a new theme (e.g., "Unit 3 — Travel").
  2. Click Add Word to open the word form. Fill in the word itself, then add translations in any of the four supported languages: French, English, German, and Italian. Also add IPA, definition, synonyms, and antonyms where possible.
  3. The English field is only needed when the source word is not English (e.g., if the main word is a French or German word, enter its English equivalent here).
  4. Use Rename Theme to reorganise without losing data, or Merge Themes to combine two.
Tip: The more fields you fill in per word (IPA, definition, synonyms, translations), the more become available for that theme. The active interface language controls which translation is shown by default in games.

Adding words via JSON import

Use the JSON or Bulk JSON tab in the Add Word panel to paste an array of word objects. Include as many language fields as you have:

{
  "word": "ephemeral",
  "ipa": "ɪˈfem.ər.əl",
  "partOfSpeech": "adjective",
  "level": "C1",
  "theme": "Time",
  "french": "éphémère",
  "english": "ephemeral",
  "german": "vergänglich",
  "italian": "effimero",
  "definition": "Lasting for only a short time.",
  "synonyms": ["fleeting", "transient"],
  "antonyms": ["permanent"]
}
Tip: Click Copy AI Prompt in the JSON tab to get a ready-made prompt for an AI assistant — it generates correctly formatted word objects including all four language fields.

Gap-fill banks power the Gap Fill activity in . Each entry is a complete sentence with one word removed; students supply the missing word.

Error banks power the Find the Error activity in . Each entry pairs an erroneous sentence with its correct version and a short pedagogical explanation.

  1. Click Error Banks in the sidebar, then open or create a file.
  2. Click Add Record. Fill in correct, erroneous, and explanation (all required). Optionally add a category (e.g., "Tenses", "Common Mistake").
  3. Use the JSON tab or Copy AI Prompt to bulk-import error pairs.

  2. Click Add Record. Enter a title to identify the entry, then fill in part1 through part8 — at least two parts required. Leave unused part fields empty.
  3. Use the JSON tab or Copy AI Prompt to bulk-import sentence entries.
  1. Click Stories to create branching or linear stories for the Choose Your Story activity.
  2. Each node has a text passage and one or more choice options linking to other nodes.
  3. Embed vocabulary words from your word banks to reinforce learning in context.
  1. Click Quizzes in the sidebar, then open or create a file.
  2. Add multiple-choice questions — mark the correct answer. Questions can include images or audio.

File Manager

Browse, rename, open, and duplicate your data files from one place. Three tabs cover recent sessions, full folder navigation, and all data & storage settings including backup and sync.

Shows your constellation maps and Board session files sorted by most recent. Use it to quickly reopen or tidy up saved work.

  • Search — type to filter files by name in real time.
  • Type filter — narrow the list to Constellation maps, Session files, or Documents (PDFs, images, sounds, books).
  • Sort — order by Newest first, Oldest first, Name A→Z, or Name Z→A.
  • Open in Board — appears on constellation map rows; click to reopen the map directly in the Board tool.
  • Rename — click Rename on any session or map row to edit the name inline. Press Enter to save, Escape to cancel. The file extension is preserved automatically.
Rename warnings: If you try to save an empty name, use forbidden characters (\ / : * ? " < > |), exceed 255 characters, or pick a name already taken in the same folder, a red warning appears below the input and the save is blocked until you fix it.

Explore any data area as a real folder tree. Navigate sub-folders, open files in external apps, duplicate or rename anything.

  • Use the target selector to choose the data area: Constellation Maps, User Data, Custom Data, Custom Wordbanks, Books, Dictations, Quizzes, Grades, Grade Sheets, or Participation logs.
  • Click a folder row to navigate into it. Click any segment of the breadcrumb at the top to jump back up.
  • Type in the search box to filter by filename at the current folder level.
  • Click ↺ Refresh to reload the list from disk after external changes.
  • Open — opens the file in the default desktop application (e.g., a PDF in your PDF viewer, an image in Photos).
  • Duplicate — copies the file in the same folder with a – Copy suffix. If that name already exists, it appends – Copy (2), – Copy (3), and so on. The new row appears immediately below the original.
  • Rename — edits the name inline with the same validation as the Recent tab (forbidden characters, length, conflict checks).

The Sync tab consolidates all storage controls in one place: where your data lives, where backups go, and how to sync between two machines.

Data folder

  • Shows the current data folder — the root location where all app files are stored.
  • Change folder… — opens a folder picker to move your data root to a different location (e.g., a OneDrive or Dropbox folder so it syncs automatically via your cloud provider).
  • Reset to default — moves the data root back to the default location inside the app directory.

Backup

  • Choose Folder — pick a destination for manual backups (a USB stick, external drive, or separate cloud folder).
  • Backup Now — copies all user data files to the backup folder immediately. The button is disabled until a backup folder is set.

Sync

  • The sync feature keeps two copies of your data folder in step — useful for working across a classroom PC and a laptop at home via a shared USB stick or network drive.
  • Choose Folder — point to the sync location.
  • Sync Now — copies files that differ between your active data folder and the sync location. Newer files win.
  • Enable Keep target up to date automatically to sync on every file save without clicking.
  • If both sides have changed the same file since the last sync, a conflict list appears. For each file choose Mine (keep your local version), Sync (take the version from the sync folder), or Skip (leave both unchanged). Use the bulk buttons to resolve all at once, then click Apply choices.

Class Plan

Design, save, and print seating plans for your classroom. Choose from three desk layouts, assign students by drag-and-drop, and randomise seats in one click. Plans sync automatically with Class Management.

LayoutShapeConfiguration
GridRectangular rows × columnsRows (1–12), Columns (1–14). Add/remove rows and columns from the editor toolbar.
U-ShapeOne top row + two side armsWidth — number of desks across the top row (also sets arm spacing). Depth — number of desks on each arm.
PodsClusters of desks arranged in a room gridPod size (rows × cols per cluster) and Room layout (how many pod rows/columns in the room).

The layout type is chosen when creating a plan and can be changed in Settings. Changing the layout clears existing seat assignments.

  • Drag-and-drop — drag a student chip from the left panel onto any empty desk cell.
  • Swap seats — drag a name from an occupied desk directly onto another desk to swap or move.
  • Unassign — click the ✕ button on a desk cell to return the student to the unassigned panel.
  • 🎲 Random — fills all seats with shuffled students. Asks for confirmation if seats are already assigned.

Placed students appear greyed out in the student panel. Unassigned students remain draggable.

  • Plans are auto-saved after every seat change (0.7 s debounce).
  • Primary storage: user/class-plans/plans.js.
  • On every save, plans are also written into user/config.js so Class Management picks them up on next load.
  • Plans created in Class Management are imported into Class Plan on startup.
  • Use 🖨 Print to produce an A4 seating chart for any layout type.
  • Mark a plan as Active to make it the default shown in Class Management's plan view.

Schedule Maker

Plans oral exam sessions where one student prepares in a separate room while the previous student presents. Handles SEN accommodation times, breaks, random or manual ordering, and produces a printable timetable.

  1. Select a Class / group from the dropdown — students are loaded from class-groups.js.
  2. Set Prep time – standard: how long a regular student needs in the preparation room (e.g., 10 min).
  3. Set Prep time – SEN: how long a student with a special education needs accommodation needs (e.g., 20 min).
  4. Set Exam time: how long each student's oral presentation lasts (same for all students).
  5. Enter Number of breaks and Break duration. Breaks are placed evenly when you click Apply and can be repositioned by dragging.
  6. Choose an Initial order — Random, Alphabetical, or Keep current (preserves the existing drag order when re-applying).
  7. Click Apply to generate the student list and schedule.
SEN flag: Students with the SEN checkbox checked in are automatically marked as SEN here. You can override this per session using the SEN toggle on each row.
  1. After clicking Apply, the Student Order list shows all students and break markers.
  2. Drag the ⠿ handle on any row to reorder students or move break markers to a different position.
  3. Toggle the SEN checkbox on any student row to enable or disable the SEN accommodation for this session only — it does not affect the saved class data.
  4. Click ✕ on a break row to remove that break, or use + Add Break to add one at the end.
  5. The schedule table below updates live as you make changes.

The schedule uses a concurrent preparation/examination model: while one student presents, the next student is already preparing in a separate room.

  • First student (in each segment, i.e., at the start or after a break): pure preparation time, then exam. No one presents while they prepare.
  • Middle students: enter the prep room the moment the previous student enters the exam room. Their exam starts when the previous student's exam ends — unless their prep time is longer than the exam time, in which case the examiner waits briefly (a ⚠ gap warning appears).
  • Last student before a break or at session end: presents with no one preparing behind them.
  • After a break: the first student in the new segment starts fresh with a full prep period — breaks are clean pauses for everyone.
Example: Exam time = 15 min, standard prep = 10 min. Student 1 prepares 08:00–08:10, presents 08:10–08:25. Student 2 prepares 08:10–08:20 (5 min wait), presents 08:25–08:40. Each slot is 15 min from Student 2 onward.
  1. The schedule table shows each student's number, name, SEN badge (if applicable), prep start time, exam start time, and exam end time. Breaks and any examiner wait gaps are shown inline.
  2. Click Print to open the browser print dialog — only the schedule table is printed (configuration and ordering panels are hidden).
  3. Click Save to save the current session (configuration + student order) as a schedule-YYYY-MM-DD-HHmm.js file in your user data folder.
  4. Click Load to open a list of previously saved sessions and restore one.
Tip: Save often — especially after setting up a complex manual order — so you can restore it if needed.

Document Editor

A dual-pane Markdown editor with live KaTeX maths rendering, custom CSS stylesheets, document and template management, PDF/DOCX export, and book text import. Write on the left; see the formatted output on the right.

The window is split into a Markdown editor (left) and a live preview (right). Drag the vertical divider to resize either pane.

Write standard GitHub Flavored Markdown with two maths extensions:

  • $…$ — inline maths (KaTeX)
  • $$…$$ — display (block) maths (KaTeX)

The nav bar is organised into hover menus:

  • File ▾ — New, Open, Save, Save As, Save All, History
  • Insert ▾ — Table, Image, Books (book text import)
  • Format ▾ — Templates, Page Layout
  • Export ▾ — PDF, DOCX, Preview
  • ⚙ Settings — snippets, keyboard shortcuts, triggers, preferences
Shortcuts: Ctrl+S saves · Ctrl+N new document · Ctrl+O open document · Esc closes any open modal.

The top panel lets you control the visual appearance of the preview. It has three parts:

  • Stylesheet picker — select a saved .css file. Use New / Save / Delete to manage stylesheet files stored in custom-data/document-editor/stylesheets/.
  • Quick Rule builder — type a selector, choose a CSS property from the dropdown, type a value, and press + Add. The rule is appended to the CSS editor and applied to the preview immediately.
  • CSS Editor ▾ — click to expand a full Monaco CSS editor showing the active stylesheet. Changes are applied live. Click Save to persist them.
Use .doc-preview as a root selector to scope your rules to the preview area, e.g. .doc-preview h1 { color: #2c3e50; }.

Documents are .md files saved under custom-data/document-editor/docs/.

  • New — clear the editor for a new document (prompts if unsaved changes exist).
  • Open — shows a list of saved documents. Click Open to load or Del to delete.
  • Save — saves under the current filename; asks for a name if the document is new.
  • Save As — always prompts for a filename.

Templates are .md files under custom-data/document-editor/templates/. Use Format ▾ to open a template into the editor or save the current content as a new template.

The Insert ▾ menu contains a Books dropdown that lists all files in custom-data/books/ (.epub, .html, .txt). Selecting a book opens a picker modal:

  • The full book text is displayed in a scrollable area.
  • Type in the search bar to find a passage — the match count is shown immediately.
  • Press Enter or the ▼ / ▲ buttons to jump to each match; the found text is highlighted by browser selection.
  • Adjust the selection with Shift+click or Shift+arrows if you need more than the matched word.
  • Click Insert selection to paste the selected text at the editor cursor.
Press Enter repeatedly to step through all matches. Shift+Enter steps backwards.

Click Export PDF in the nav bar. The current preview — including KaTeX maths, custom CSS, and default document styles — is rendered and the system save dialog opens. Choose a folder and filename to save the PDF.

The PDF captures exactly what is visible in the preview pane, so apply your stylesheet before exporting.
